# 有，半天上线了一个完整网站，再用 AI 搭了一个开源项目——但关键不是 AI 写了多少代码

先说结论：**AI 编程工具确实能让你快速开发程序，但"快速开发"和"快速开发出能用的东西"之间，隔着一条叫"工程判断力"的鸿沟。**

我是一个写了十几年代码的老程序员。过去一年，我用 AI（Claude Code、Codex）做了两个完整项目，都是从零到上线。今天不吹不黑，只说真实经历。

---

## 案例一：半天上线 agentskill.work

去年有个想法：做一个网站，把 GitHub 上 Agent/Claude Skill 相关的仓库抓出来，做成可浏览、可搜索的目录站。

**从想法到上线，半天。**

技术栈是 Next.js + FastAPI，AI 写了绝大部分代码。但重点不在"代码是 AI 写的"——这年头让 AI 写个 CRUD 谁不会。

重点在于：**从抓取、限流、定时任务，到 Nginx、证书、Docker Compose 一键拉起，再到 Umami 埋点统计——全链路都是 AI 做的。**

但别误会，这不是"AI 全自动"。我这半天最重要的工作，不是敲代码，而是盯住几个关键节点：

### 1. 限流：否则 GitHub API 一限你就死了

AI 默认会直接前端请求 GitHub API。我第一件事就是否掉这个方案：**必须有服务端抓取的节奏控制，必须有缓存，失败要能降级到上一次结果。**

### 2. 定时任务：用 Celery 不是炫技，是可控

很多 AI 会下意识给你塞个 cron 或者前端定时刷。看起来省事，后面全是债。我直接用 Celery 定时任务——失败可重试、可追踪、节奏可控。

### 3. 技术选型：AI 想用 Caddy，我选 Nginx

AI 选型追求"配置看起来优雅"。老程序员选型追求"出了事能扛、扛得住、扛得明白"。Nginx 的生态、资料、经验密度，不是 Caddy 能比的。

### 4. 数据闭环：第一天就搭 Umami

很多人做站：先功能、再增长、最后再说统计。我上线同时就把 Umami 搭好。**没有统计，你连迭代方向都是盲的。**

## 案例二：Orion——AI 写的开源工程范本

另一个更大的项目：**Orion，统一通知网关**。几乎所有代码都由 AI 生成，我主要做的是 glue（拼合、调整）和规范制定。

这个项目让我确信了一种新的开发模式：

- **AI 是施工队，我是总工 + 质检员**
- AI 负责 scaffold（FastAPI + SQLAlchemy + Next.js + Docker Compose），速度无人能敌
- 我负责写 `AGENTS.md` 作为项目"宪法"，规定目录结构、提交规范、分支策略
- 每次和 AI 的对话存到 `agents_chat/`，保证开发过程可审计、可复现

### 从"能跑"到"能运"的提交路径

这是我总结的一条金线：

1. **文档先行**：先写设计文档、接口草案
2. **竖切 MVP**：从 API → 单一适配器 → 存储 → 简单可观测
3. **可靠性基线**：幂等键、重试退避、死信队列、限流、熔断
4. **安全加固**：敏感配置加密、审计日志、RBAC
5. **可观测 & 运维**：投递生命周期追踪、仪表盘、失败告警

AI 负责 1-2 的大头，3-5 我强制落实。这样出来的东西不是"能跑的玩具"，而是"能上线的系统"。

## 所以，AI 编程工具到底能不能快速开发程序？

**能，但有条件。**

| 维度 | AI 擅长 | 人必须补 |
|------|---------|---------|
| 速度 | scaffold 极快，一句话出页面 | — |
| 功能 | CRUD、UI、接口调用 | 可靠性（幂等、熔断、退避） |
| 部署 | Docker Compose、基础配置 | 证书续期、持久化、故障排查 |
| 一致性 | 有规范就能统一 | 没规范就飘 |
| 深度 | 浅层功能很强 | 安全、可观测、运维需要经验 |

**一句话总结：AI 给了速度，人补上深度。**

如果你有工程经验，AI 编程工具是倍增器——半天能干以前一周的活。

如果你没有工程经验，AI 编程工具是"能跑的幻觉制造机"——你以为上线了，其实埋了一堆雷。

---

## 给想试的人几个实操建议

1. **先用 AI 写，但写完自己必须 review**——至少看懂它在干什么
2. **技术选型自己定**——别让 AI 替你选框架，它没踩过你将要踩的坑
3. **第一天就加监控**——哪怕只是一个 Umami 或简单的日志
4. **把开发过程存下来**——Prompt、AI 输出、你的修改，都记录。将来你会感谢自己
5. **从小项目开始**——先用 AI 做一个能上线的小工具，积累信心和经验

AI 编程工具不是魔法，是杠杆。杠杆放大的是你已有的能力——包括你的判断力、经验和品味。
